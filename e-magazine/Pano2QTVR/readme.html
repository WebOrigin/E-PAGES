Pano2QTVR for Windows - Quicktime VR Converter

Commandline Parameter

PanoQTVR [Options] <inputfiles>

If you call Pano2QTVR without parameters two dialogboxes ask for the input and the output file. If you have selected the files the program starts to convert the image.

Possible options are:
-e only extract the cubefaces as JPEG
-x only extract the cubefaces as TIFF
-a assemble the JPEG cubefaces
-ac compress TIFF cubefaces and assemble the resulting JPEG
-r assemble cubefaces to an equirectlinear image
-t Create a thumbnail image
-h Create a HTML Template
-dosup use the same number system as the DosUP Tools by Eric Gerds
-s Silent. don't show the progress dialog
-k Keep the temp cubeface files
-f <facename> default name for the cubeface files. The resulting files are <facename>_0.tif - <facename>_5.tif or with -dosup <facename>11.tif to <facename>66.tif
-c <file> additional config file. You can have as many config files as you like.
-o <.mov file> output filename. You can use $f for the filename of the input filename

Configuration file

Pano2QTVR looks at several places for config files. At first it trys to locate the pano2qtvr.cfg in the path of the executable. Then it tries to load the pano2qtvr.cfg in the current working directory and the directory of the input image. After that, all specified files from the commandline are used.

The config file is a plain text file where each line can hold a parameter followed by an = and the value. Comments start with a # and can only start in a new line! You can use the provided pano2qtvr_tmpl.cfg as a starting point. Just rename it to pano2qtvr.cfg

Each config file can have the same parameters. Only the last value for each parameter is used. So it is possible to have structured default values.

For example you can set you copyright information and default stitcher settings in the file within the path of the executable. In the file within the working path you set parameters like the title of the movie or a special tile size.

You can also have a library of config files for special settings like "small.cfg" and "large.cfg" for often used parameters or you can make config files for different external compressors like "imagemagick.cfg" or "irfanview.cfg"

User Data

The userdata parameters are string values for additional information, that is stored within the movie. You can use the following variables within each userdata field

$f Full path and filename of the input file
$n filename of the input file
$o Full path and filename of the output file
$d Date and time of the input file. The format can be specified in userdata.date.format

Hotspots

Hotspots are links within the QTVR movie. The hotspot track is just another cube drawn with special colors. Each color is another hotspot. The default format for the track is PNG. You can specify a title and a target URL for each color.

To open a movie within the same player you have to specify the URL in the form of

A<newMovie.mov> T<myself> E<pan=123.0 tilt=30.0 fov=70.3>
